- **MM ทั้งโมดูล — STORAGE vs company table split (เจอซ้ำ 3 ไฟล์แล้ว)**: ก่อนแตะไฟล์ `create_temp_*` ใดๆ ในโมดูล MM **ต้องเช็คค่า system param `CALCULATE_COST` ก่อนเสมอ** (`SELECT KEYVALUE FROM systemparametersystemparameterparamitem WHERE KEYNAME='CALCULATE_COST'`). ถ้าเป็น `STORAGE` → ต้องอ่าน/เขียนตาราง `poststockpoststock*` (storage-level) เท่านั้น — ไฟล์ที่ยัง hardcode `poststockcompanypoststockcompany*` (company-level) จะได้ 0 แถวเสมอบน DB ที่ตั้ง STORAGE mode (เจอใน `create_temp_product_material.jsp` เก่า, `create_temp_inventorystatus_ac_sumcost.jsp` เดิมใน MM/I/06/0 — ทั้งคู่ยังไม่ได้แก้ถ้าถูกเรียกใช้จริง). ไฟล์ที่เขียนถูกแล้วเป็นตัวอย่างอ้างอิงได้: `create_temp_inventorystatus_ac_sumcost_new.jsp` (MM/G/01/0) มี branch check ครบ. **แนะนำ**: ก่อนทำรายงาน MM ตัวถัดไป ให้ grep หาไฟล์ `create_temp_*` ที่ใช้ `poststockcompany` แบบไม่มี branch check ก่อน แทนรอให้ user เจอทีละตัว
- **ตารางธุรกรรม MM มี column คู่**: `poststockpoststockmovement` มีทั้ง `dr`/`cr` (บัญชีคู่, มักไม่ถูกเติมค่าโดยธุรกรรม GR/GI) และ `qty` (สุทธิมีเครื่องหมาย, มีค่าจริง) — ก่อนเขียน `sum()` query ใหม่ ให้รัน `SELECT txtype,sum(qty),sum(dr),sum(cr) FROM poststockpoststockmovement GROUP BY txtype` เช็คก่อนว่า column ไหนถูกใช้จริง อย่าเดาจากชื่อ column
- **Drilldown "คงเหลือ" (running balance) ห้าม map ไปฟิลด์ยอดยกมาเฉยๆ** — ต้องคำนวณด้วย SQL window function (`SUM(...) OVER (PARTITION BY material ORDER BY วันที่ ROWS UNBOUNDED PRECEDING)`) ให้ตรงสูตร jasper variable (ดู `.jrxml` หา `<variable name="balance_*">`) · **`getsumary()` (shared JS, `assets/js/index.js`) ตอนนี้แก้แล้ว** — มี `RUNNING_BALANCE_FIELDS` array กำหนด field ที่ต้อง sum แบบ "เอาแถวสุดท้ายต่อกลุ่ม (group key=`materialcode`) มาบวกกัน" แทน sum ทุกแถวตรงๆ (ปัจจุบันมี `balance_amount`,`balance_quantity` — เพิ่ม field ใหม่เข้า array ได้ถ้าเจอ pattern เดียวกันที่รายงานอื่น) ใส่ field พวกนี้เข้า `level1_filtersummary` ได้แล้วปกติ **แต่ field ที่เป็นราคา/หน่วยหรือ ratio อื่นๆ (เช่น `*_cost_per_unit`) ยังห้ามใส่เข้า filtersummary เด็ดขาด** ไม่มีวิธี sum ที่มีความหมายแม้จะ group แล้วก็ตาม
