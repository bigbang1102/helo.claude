## Module patterns (กฎระดับโมดูล — เติมเมื่อเจอซ้ำ)
- **pos/changeposoutbounddelivery/cposob**: ฟอร์มโมดูลนี้หลายตัวมีแต่ `.jasper` compiled ไม่มี `.jrxml` source ใน repo — เจอ resource-bundle key (`$R{label.gen.*}`) ฝังในไบนารีได้ด้วย grep ASCII ของ key name (แต่ grep ข้อความไทยตรงๆ ใน `.jasper` ไม่เจอ เพราะ object stream ถูก compress/serialize) · ก่อนแก้ label ใดๆ ต้อง grep key name ข้ามทั้ง repo ก่อนเสมอ (`label.gen.xxx` มักใช้ร่วมข้ามรายงานหลายตัวคนละโมดูล เช่น POSG reportdefault)
- **py/pv/pv (E-PV-xx family)**: ฟอร์มกลุ่มนี้กำลัง migrate จาก multi-file-per-page (`main_page_pv` เลือกไฟล์ตาม `page_count`/`row_count`) ไปเป็น query-เดียว-รวม + self-pagination ในตัว (`<break>` + `pageFooter`/`lastPageFooter`) — **ก่อน clone ไฟล์ในกลุ่มนี้เพื่อทำ "หน้า/กรณีถัดไป" ต้องอ่าน `<detail>`/`<break>`/`pageFooter`/`lastPageFooter` ของไฟล์ต้นทางให้ครบก่อนเสมอ** อย่าเชื่อว่า pattern เก่า (1 ไฟล์ = 1 หน้า) ยังใช้ได้ · ไฟล์ `.jasper` เก่าที่ไม่มี `.jrxml` เหลือ แต่ query SQL เป็น plaintext ฝังอยู่ ดึงออกมาได้ด้วย regex กวาดหาข้อความ `select`/`SELECT` (ใช้ PowerShell `[System.Text.Encoding]::UTF8.GetString` อ่านไฟล์เป็น byte array)
